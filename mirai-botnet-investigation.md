# Investigation: Mirai Botnet Scanning Activity

## Overview
During routine monitoring of network telemetry within the SIEM platform, multiple intrusion detection alerts indicated 
possible botnet scanning activity targeting IoT devices. The alerts were generated by the Suricata intrusion detection 
system and suggested exploitation attempts against DVR systems commonly targeted by Mirai botnets.

## Monitoring Environment

Network telemetry was analyzed using the following tools:

- Elastic Stack (SIEM)
- Kibana dashboards for alert visualization
- Suricata IDS for network intrusion detection

Threat intelligence sources used during investigation included:

- VirusTotal
- GreyNoise
- AbuseIPDB
- Cisco Talos Intelligence

## SIEM Dashboard

The following dashboard was used to monitor Suricata alerts and identify anomalous network activity patterns.

## Detection

While reviewing Suricata alerts within the Kibana dashboard, several IDS signatures were triggered:

ET SCAN Mirai Variant User-Agent  
ET SCAN JAWS Webserver Unauthenticated Shell Command Execution  
ET EXPLOIT MVPower DVR Shell UCE  

These signatures are commonly associated with Mirai botnet scanning activity.

## Investigation Process

1. Identified the source IP address generating the alerts.
2. Reviewed traffic patterns in the SIEM to determine frequency and timing of the events.
3. Queried threat intelligence sources to determine reputation and previous reports associated with the IP address.
4. Reviewed related alerts within the SIEM to determine whether the activity was isolated or part of a broader
   scanning pattern.

## Threat Intelligence Findings

Investigation of the source IP revealed:

- Multiple abuse reports on AbuseIPDB
- Indicators of automated scanning behavior
- Similar activity reported in GreyNoise datasets associated with internet-wide scanning

These indicators suggested that the source host was likely part of a botnet conducting automated reconnaissance.

## Analysis

Mirai botnets commonly target vulnerable IoT devices such as:

- DVR systems
- routers
- IP cameras

The IDS alerts observed indicated attempts to exploit services commonly exposed by DVR devices. The frequency and 
pattern of the alerts suggested automated scanning activity rather than targeted intrusion.

## Conclusion

Based on the IDS signatures, IP reputation data, and observed traffic patterns, the activity was assessed to be part 
of automated botnet scanning activity associated with Mirai-style malware.

## Response

The activity was documented and submitted as an incident investigation ticket for escalation and review by monitoring 
personnel.

Continued monitoring was performed to determine whether additional related traffic would occur.
